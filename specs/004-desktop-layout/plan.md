# Implementation Plan: Раскладка для широкого экрана

**Branch**: `main` | **Date**: 2026-07-22 | **Spec**: [spec.md](./spec.md)

**Input**: Feature specification from `/specs/004-desktop-layout/spec.md`

## Summary

На экранах шире 1024 px горизонтальная лента чипов заменяется вертикальным
оглавлением сбоку, колонка текста остаётся ограниченной по ширине. На узких
экранах не меняется ничего.

Это **первая с фичи 001 правка кода приложения** — предыдущие две фичи были
чисто контентными. Отсюда главная забота плана: не сломать то, что уже принято
врачом и работает в проде. Мобильная раскладка остаётся веткой по умолчанию
на всех уровнях, вплоть до случая «ширину определить не удалось».

Попутно зафиксирован уже устранённый дефект: лента чипов не прокручивалась
мышью, из-за чего часть секций была недостижима.

## Technical Context

**Language/Version**: без изменений — TypeScript 5.9 / Vue 3.5 / Tailwind 4 / Node 22.

**Primary Dependencies**: ничего нового. `matchMedia` — браузерный API,
не зависимость.

**Storage**: не затрагивается. Файлы в `src/data/` не меняются.

**Testing**: существующий Vitest + новый тест раскладки. Тесты навигации
фичи 001 обязаны пройти **без единой правки** — это и есть проверка SC-003.

**Target Platform**: тот же мобильный браузер плюс десктопный браузер.

**Project Type**: изменение представления в принятом SPA.

**Performance Goals**: одна подписка на `matchMedia` на приложение; рендерится
один вариант навигации вместо двух. Влияния на размер бандла и офлайн нет.

**Constraints**: телефон важнее монитора. При любом сомнении выигрывает
мобильное поведение.

**Scale/Scope**: один новый composable, правка трёх компонентов, один новый тест.

## Constitution Check

*GATE: пройден до реализации, перепроверяется после.*

| Принцип | Как соблюдается | Проверяется чем |
|---------|-----------------|-----------------|
| I. Справочник, а не советчик | Фича только о раскладке. Ни вычислений, ни интерпретации не добавляется | Ревью диффа |
| II. Запрет на генерацию контента (NON-NEGOTIABLE) | Клинического текста фича не касается вовсе: `src/data/` не в диффе | `git diff --stat`, `verify:transcription` |
| III. Данные отдельно от кода | Правка ровно обратная обычной — только код, ноль данных. Разделение не нарушается | `git diff --stat` |
| IV. Полный офлайн | Сетевых обращений не вводится; `matchMedia` работает офлайн | Сценарий офлайна в quickstart |
| V. Прослеживаемость | Поля протокола и их отображение не трогаются | `validate:data`, существующий тест прослеживаемости |
| VI. Приватность | Ширина окна никуда не отправляется и не сохраняется | Ревью |
| VII. Работоспособность web-сборки | `build`, `test`, `type-check` в DoD | CI |

**Нарушений нет.** Отдельно отмечу принцип VII: фича трогает компонент, через
который проходит вся навигация по протоколу. Поэтому в DoD вынесено отдельным
пунктом требование, чтобы тесты навигации прошли без правок — правка теста
здесь была бы способом не заметить регрессию.

**Structure Decision**: новый composable, а не логика внутри компонента.
Раскладку знают три места (навигация, экран протокола, каркас), и три
независимых подписки на один медиазапрос разъехались бы при правке порога.

## Ключевые решения

Подробно — в [research.md](./research.md). Коротко:

| # | Решение | Чем платим |
|---|---------|------------|
| R1 | Один порог 1024 px | На планшете остаётся мобильная раскладка |
| R2 | Вариант навигации выбирается в коде, в DOM он один | Появляется зависимость от `matchMedia` |
| R3 | Не определили ширину → мобильная раскладка | На экзотическом окружении монитор недоиспользуется |
| R4 | В обвязку регистрируется только горизонтальная лента | Требует внимания при правке разметки навигации |
| R5 | Колонка текста ограничена | Часть ширины монитора остаётся полями |
| R6 | Дефект прокрутки ленты починен отдельно | — |

## Риски

| Риск | Почему серьёзен | Что делаем |
|------|-----------------|------------|
| Регрессия мобильной навигации | Это единственный сценарий цейтнота, и он уже принят врачом | Мобильная ветка — дефолт везде; тесты навигации не правим |
| Отступ якоря разъедется | Клик уведёт секцию мимо верха экрана, и это заметно не сразу | R4: сайдбар не регистрируется в обвязке; проверяется вручную в quickstart |
| Подсветка активной секции сломается на десктопе | Наблюдатель настроен на высоту обвязки, а она изменилась | Общий механизм для обоих представлений, отдельный сценарий приёмки |

## Definition of Done

- [ ] `npm run test` зелёный, и **тесты навигации фичи 001 не правились**
- [ ] `npm run type-check`, `npm run build` зелёные
- [ ] `npm run validate:data`, `npm run verify:transcription` зелёные
      (не должны были затронуться — проверяем именно это)
- [ ] `git diff --stat` не показывает изменений в `src/data/`
- [ ] На ширине 1280 px видно оглавление со всеми секциями, переход одним кликом
- [ ] На ширине ~380 px поведение неотличимо от прежнего
- [ ] Отступ якоря корректен в обеих раскладках
- [ ] Лента чипов прокручивается мышью в узком окне
- [ ] Офлайн на прод-сборке работает как раньше

# Quickstart: приёмка раскладки для широкого экрана

Ссылки: [spec.md](./spec.md), [plan.md](./plan.md), [research.md](./research.md).

## Предпосылки

Всё из [quickstart фичи 001](../001-protocol-viewer/quickstart.md) (Node 22, `npm ci`).
Исходный документ владельца для этой фичи НЕ нужен: контента она не касается.

## Команды

| Команда | Что делает |
|---------|------------|
| `npm run test` | Vitest, включая раскладку навигации |
| `npm run type-check`, `npm run build` | сборка не сломана |
| `npm run dev` | ручная проверка раскладок |
| `npm run preview` | прод-сборка, нужна для офлайна |

## Сценарий 1. Оглавление на широком экране (US1, FR-001/004; SC-001/002)

1. Открыть памятку по ВСТЭ в окне шириной 1280–1440 px.
2. Ожидаемо: слева — оглавление, все **тринадцать** пунктов видны сразу,
   прокручивать оглавление не нужно.
3. Ожидаемо: горизонтальной ленты чипов на экране нет вовсе.
4. Кликнуть в последний пункт. Ожидаемо: переход к секции, пункт помечен
   активным, адрес страницы не изменился.
5. Прокрутить документ вниз до конца. Ожидаемо: оглавление осталось на экране.

---

## Сценарий 2. На телефоне ничего не изменилось (US2, FR-002; SC-003)

1. Открыть ту же памятку на ширине ~380 px.
2. Ожидаемо: горизонтальная лента чипов, как раньше; оглавления сбоку нет.
3. Пройти сценарии навигации фичи 001: тап по чипу, подтягивание уехавшего
   чипа, подсветка активной секции при прокрутке.
4. Ожидаемо: поведение неотличимо от прежнего.
5. `npm run test`. Ожидаемо: `tests/section-nav.spec.ts` зелёный **и не
   правился** — если он потребовал правки, это регрессия, а не устаревший тест.

---

## Сценарий 3. Отступ якоря в обеих раскладках (FR-010; R4)

Самая незаметная поломка этой фичи: если оглавление попадёт в расчёт
закреплённой обвязки, переход по клику будет уводить секцию слишком низко.

1. На широком экране кликнуть по пункту в середине оглавления.
2. Ожидаемо: заголовок секции встал сразу под шапкой и панелью поиска, а не
   на середине экрана.
3. На узком экране повторить тап по чипу из середины ленты.
4. Ожидаемо: заголовок секции встал сразу под лентой чипов.
5. Контроль в консоли: `getComputedStyle(document.documentElement)
   .getPropertyValue('--section-nav-height')` — на широком экране должен быть
   `0px`, на узком — фактическая высота ленты.

## Сценарий 4. Лента чипов работает мышью (US3, FR-011…FR-014; SC-004)

1. Сузить окно браузера на компьютере так, чтобы появилась лента.
2. Ожидаемо: у ленты видна полоса прокрутки.
3. Навести курсор на ленту, покрутить колесо. Ожидаемо: лента едет по
   горизонтали, последний чип достижим.
4. Докрутить ленту до края и продолжить крутить. Ожидаемо: начинает
   прокручиваться страница — лента не «съедает» жест.
5. На телефоне: полоса прокрутки не занимает высоту, лента свайпается пальцем.

---

## Сценарий 5. Текст остаётся читаемым (US4, FR-005; SC-005)

1. Открыть памятку по ТЛТ на мониторе 1920 px, найти подпункт 28.1 (~900 знаков).
2. Ожидаемо: строка не растянута во всю ширину экрана; свободное место ушло
   в оглавление и поля.

---

## Сценарий 6. Переключение раскладки на лету (edge case)

1. Открыть протокол и медленно менять ширину окна через 1024 px.
2. Ожидаемо: раскладка переключается один раз, без мерцания.
3. Ожидаемо: подсветка активной секции не теряется, отступ якоря остаётся верным.

---

## Сценарий 7. Контент не тронут (FR-015; SC-006)

1. `git diff --stat`. Ожидаемо: в `src/data/` изменений **нет**.
2. `npm run validate:data` и `npm run verify:transcription`. Ожидаемо: зелёные —
   фича не должна была их затронуть, и это проверяется явно.

---

## Сценарий 8. Офлайн (FR-017)

1. `npm run build && npm run preview`, дождаться установки SW.
2. Devtools → Network → Offline → перезагрузка.
3. Ожидаемо: обе памятки открываются, обе раскладки работают, внешних
   запросов ноль.

### Состояние проверки

Раскладки, число пунктов оглавления и отступ якоря проверены в headless-браузере
на 1440×900 и 390×844, а не только тестами: снимки экрана и замеры геометрии
делались на прод-сборке. Отступ якоря измерен в обе стороны прокрутки —
и сверху вниз, и снизу вверх.

Не проверено машинно и остаётся владельцу: ощущение от прокрутки ленты колесом
мыши и офлайн на реальном устройстве.

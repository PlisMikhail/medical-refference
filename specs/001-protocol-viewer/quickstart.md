# Quickstart: проверка фичи «Просмотрщик протоколов»

Прогоняемые сценарии, доказывающие соответствие спеке. Ссылки:
[spec.md](./spec.md), [data-model.md](./data-model.md),
[contracts/](./contracts/).

## Предпосылки

- Node 22 LTS (`.nvmrc` в корне), npm
- Установка: `npm ci` (или `npm install` при первом запуске)

## Команды

| Команда | Что делает |
|---------|------------|
| `npm run dev` | dev-сервер с dev-валидацией данных |
| `npm run validate:data` | схемы + кросс-проверка реестра и файлов |
| `npm run test` | Vitest smoke-тесты |
| `npm run build` | прод-сборка (web + SW) |
| `npm run preview` | локальный прогон прод-сборки (нужен для проверки офлайна) |

## Сценарии проверки

### 1. Список и просмотр протокола (US1, FR-001/002/005; SC-003)

1. `npm run dev`, открыть приложение (mobile viewport ~380px в devtools).
2. Подтвердить дисклеймер (первый запуск).
3. Ожидаемо: главный экран со списком из `index.json` — у демо-протокола видны
   название, версия, дата источника.
4. Открыть демо-протокол. Ожидаемо: метаданные (version, sourceDocument,
   sourceDate, lastReviewed) видны; все шесть типов блоков v1 отрендерены.

### 2. Видимый фоллбэк неизвестного блока (FR-006)

1. Во фикстуре/демо-протоколе временно добавить блок
   `{"type": "totally-new-block"}`.
2. Ожидаемо: на его месте — заметная плашка «неподдерживаемый тип блока:
   totally-new-block»; dev-консоль — предупреждение; остальной протокол
   отрендерен. Откатить правку.

### 3. Навигация и маркировка секций (US2, FR-003/004; SC-001/002)

1. В открытом протоколе: sticky-табы видны и после прокрутки в конец.
2. Тап по табу — переход к секции (один тап). До любой секции от запуска
   приложения — ≤3 тапа (протокол → таб).
3. Секции показаний / абсолютных / относительных противопоказаний мгновенно
   различимы цветом (токены темы), тач-таргеты ≥44px, тема тёмная.

### 4. Чек-лист не персистится (FR-007)

1. Отметить пункты чек-листа в демо-протоколе.
2. Вернуться на главный экран, снова открыть протокол.
3. Ожидаемо: отметки сброшены; никаких сумм/итогов нигде не показано.

### 5. Поиск по открытому протоколу (US3, FR-008; SC-005)

1. Открыть поиск в протоколе, ввести строку из демо-контента в ДРУГОМ регистре.
2. Ожидаемо: совпадения подсвечены, переход между ними работает, результат
   появляется мгновенно (<1 с).
3. Ввести абракадабру. Ожидаемо: явное «ничего не найдено».

### 6. Дисклеймер (US4, FR-009; SC-006)

1. В devtools очистить localStorage → перезагрузить.
2. Ожидаемо: дисклеймер до любого контента; без подтверждения контент
   недоступен.
3. Подтвердить → перезагрузить. Ожидаемо: дисклеймер не показан.

### 7. Валидация данных (FR-011; SC-007)

1. Сломать демо-протокол (удалить `lastReviewed` или заменить `items` на
   строку).
2. `npm run validate:data` — ожидаемо: ошибка с именем файла, путём до поля и
   описанием; `npm run dev` — читаемая ошибка в оверлее (не белый экран).
3. Починить. Ожидаемо: обе проверки зелёные. Рассинхрон `version` в
   `index.json` против файла протокола также даёт ошибку кросс-проверки.

### 8. Полный офлайн (FR-013/015; SC-004; DoD)

1. `npm run build && npm run preview`, открыть, дождаться установки SW
   (Application → Service Workers в devtools).
2. Devtools → Network → Offline → полная перезагрузка.
3. Ожидаемо: приложение открывается, список и протокол работают, поиск и
   чек-лист работают; сетевых запросов к внешним доменам — ноль (вкладка
   Network пуста от внешних хостов и при онлайне).

### 9. Обновление с явным уведомлением (FR-015)

1. При запущенном `npm run preview` изменить видимый текст (не контент
   протокола), пересобрать `npm run build`.
2. Перезагрузить вкладку. Ожидаемо: баннер «Доступно обновление» с кнопкой;
   без нажатия версия остаётся старой; по нажатию — новая.

### 10. Деплой (DoD)

1. Push в `main` → GitHub Actions зелёный (validate:data → test → build →
   deploy).
2. Открыть страницу на GitHub Pages: приложение работает, deep-link
   `#/protocol/demo-protocol` открывается напрямую; после первого открытия —
   сценарий 8 на телефоне.

## Definition of Done (из вводных владельца)

- [x] Web-сборка работает в браузере (сценарии 1–7)
- [x] Полный офлайн после первой загрузки (сценарий 8)
- [x] Валидация данных проходит (сценарий 7, зелёное состояние)
- [x] Медицинского контента в компонентах нет (ревью + грep по `src/components`)
- [x] README актуален (установка, команды, как добавить протокол)

# Data Model: Просмотрщик клинических протоколов

Канонический формат — JSON Schema в [contracts/](./contracts/). Здесь — обзор
сущностей, правил валидации и состояний. TS-типы (`src/types/protocol.ts`)
зеркалят схему.

## Сущности

### ProtocolIndex — реестр протоколов (`src/data/protocols/index.json`)

| Поле | Тип | Обязательное | Назначение |
|------|-----|--------------|------------|
| `protocols` | `IndexEntry[]` | да | Упорядоченный список: порядок в файле = порядок на главном экране |

### IndexEntry — запись реестра

| Поле | Тип | Обязательное | Назначение |
|------|-----|--------------|------------|
| `id` | string, `^[a-z0-9][a-z0-9-]*$` | да | Ключ протокола; файл протокола — `<id>.json` |
| `title` | string | да | Название на главном экране |
| `version` | string | да | Версия протокола (FR-001) |
| `sourceDate` | string, ISO date | да | Дата источника (FR-001) |

Денормализация (`title/version/sourceDate` дублируются в файле протокола) —
осознанная: главный экран не грузит все файлы. Рассинхрон исключается
кросс-проверкой в `validate:data` (см. «Инварианты»).

### Protocol — документ протокола (`src/data/protocols/<id>.json`)

| Поле | Тип | Обязательное | Назначение |
|------|-----|--------------|------------|
| `id` | string | да | Совпадает с именем файла и записью реестра |
| `title` | string | да | Заголовок экрана протокола |
| `version` | string | да | Прослеживаемость (конституция V) |
| `sourceDocument` | string | да | Название/реквизиты исходного документа |
| `sourceDate` | string, ISO date | да | Дата исходного документа |
| `lastReviewed` | string, ISO date | да | Дата последней ручной проверки врачом |
| `sections` | `Section[]`, ≥1 | да | Содержимое |

### Section — секция протокола

| Поле | Тип | Обязательное | Назначение |
|------|-----|--------------|------------|
| `id` | string, `^[a-z0-9][a-z0-9-]*$` | да | Якорь навигации (уникален в протоколе) |
| `title` | string | да | Название в табе и заголовке секции |
| `kind` | enum: `default` \| `inclusion` \| `exclusion-absolute` \| `exclusion-relative` | нет (default: `default`) | Цветовая маркировка секции (FR-004) |
| `blocks` | `Block[]` | да | Содержимое секции |

### Block — discriminated union по `type`

Общее правило: `type: string` обязателен. Известные типы v1 валидируются
строго; блок с неизвестным `type` **валиден** (контрактная опора видимого
фоллбэка, FR-006), dev-валидация выдаёт предупреждение.

| `type` | Поля | Примечания |
|--------|------|------------|
| `text` | `body: string` | Абзацы через `\n\n` |
| `warning` | `body: string` | Акцентная плашка-предупреждение |
| `timer-note` | `body: string`, `label?: string` | Статическая плашка временного окна; таймера/отсчёта НЕТ (решение спеки) |
| `criteria-list` | `kind: inclusion \| exclusion-absolute \| exclusion-relative`, `items: string[]` (≥1) | Цвет маркера пункта — токен `kind`; неизвестный `kind` в данных → рендер как нейтральный список с пометкой (edge case спеки) |
| `dosage-table` | `columns: string[]` (≥1), `rows: string[][]`, `title?: string` | Все ячейки — строки; никакой арифметики (конституция I) |
| `checklist` | `items: string[]` (≥1) | Интерактивные отметки; состояние эфемерно (FR-007) |

## Инварианты (проверяются `scripts/validate-protocols.mjs`)

1. `index.json` валиден против `index.schema.json`; каждый `<id>.json` — против
   `protocol.schema.json`.
2. Биекция реестра и файлов: каждая запись реестра имеет файл
   `src/data/protocols/<id>.json`; каждый файл протокола (кроме `index.json`)
   имеет запись в реестре. Нарушение — ошибка сборки.
3. Согласованность метаданных: `title`, `version`, `sourceDate` в записи
   реестра строго равны полям файла протокола. Нарушение — ошибка
   (защита прослеживаемости, конституция V).
4. `id` внутри файла равен имени файла; `section.id` уникальны в протоколе.
5. `lastReviewed` ≥ `sourceDate` не требуется (проверка дат — только формат).

## Клиентские состояния (не персистятся, кроме дисклеймера)

| Состояние | Где живёт | Жизненный цикл |
|-----------|-----------|----------------|
| Подтверждение дисклеймера | `localStorage`, ключ `med-helper:disclaimer-accepted`, значение — ISO-дата подтверждения | Постоянно; очистка данных браузера → дисклеймер показывается снова (допущение спеки) |
| Отметки чек-листов | reactive Map в `useChecklistState` (ключ: `sectionId:blockIndex`) | Сбрасываются при unmount экрана протокола; никаких итогов не вычисляется |
| Поисковый запрос и совпадения | локальный ref в `ProtocolView` | Живёт, пока открыт протокол |
| Активная секция (подсветка таба) | IntersectionObserver в `ProtocolSectionNav` | Производное от прокрутки |

## Расширение модели (правила для будущих фич)

- Новый тип блока: ветка в `protocol.schema.json` → вариант union в
  `types/protocol.ts` → компонент в `components/blocks/` → строка в
  `registry.ts`. Старые данные остаются валидными; старый код на новых данных
  показывает фоллбэк, а не ломается.
- Новое поле метаданных протокола: добавляется в схему как optional (или
  required — тогда одним коммитом с обновлением всех JSON).
- Интерактивные фичи с логикой (например, ТЛТ-калькулятор суммы баллов) — не
  расширение этой модели, а отдельный view/модуль со своей спекой; модель
  контента остаётся декларативной.
